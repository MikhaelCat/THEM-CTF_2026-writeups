# Last dance

## Задание

> A virtualized and packed Lua malware loader was recovered from a compromised CI worker. Reverse the custom VM and extract the real flag.

Формат флага: `THEM?!CTF{...}`

## Что было дано

- `n00t.lua`

Снаружи файл выглядел как виртуализованный Lua-чекер с кастомной VM, но это в основном был декой.

## Первичная распаковка

Внешний скрипт оказался обычной оберткой над встроенным blob:

1. Вытащить большой base64-блок из `n00t.lua`.
2. Декодировать его.
3. Прогнать результат через XOR с зашитым ключом.
4. Сохранить результат как `payload_stage1.luac`.

На выходе получаем chunk с Lua 5.1 bytecode.

## Анализ stage 1

Так как подходящего дизассемблера Lua 5.1 в окружении не было, байткод пришлось разбирать вручную.

Ключевые прототипы были такие:

- `root.15` - финальный валидатор флага
- `root.17` - интерпретатор кастомной VM
- `root.8` - helper, который снимает обертку `THEM?!CTF{...}`
- `root.9` - `root.14` - вспомогательные раннеры и декои

## Главное наблюдение

Сначала казалось, что правильный путь это полный реверс VM.
После восстановления привязки closure-ов стало понятно, что реальный флоу другой:

- `root.15` проверяет, что кандидат является строкой
- проверяет общую длину `48`
- проверяет префикс `THEM?!CTF{`
- проверяет завершающую `}`
- потом берёт внутреннюю часть через `candidate:sub(11, -2)`

То есть неизвестный core имеет длину `37`.

## Почему VM была ложным следом

В байткоде действительно была кастомная VM:

- таблицы opcode-ов
- описания памяти `text`, `data`, `heap`
- host API вроде `open`, `read`, `mmap`, `socket`
- дополнительная обвязка через `coroutine`, `xpcall`, `__gc` и metatable

Но реальная проверка флага не требовала полноценного реверса VM.

## Настоящая логика проверки

Полезная часть сводилась к системе линейных уравнений по модулю `257`.

Из chunk-а удалось восстановить две функции преобразования:

```lua
f1(v, i) = ((v - 91) - i) * 121 % 257
f2(v, i) = ((v - 91) - i * 3) * 121 % 257
```

Также из корневого chunk-а были собраны:

- матрица коэффициентов `37 x 37`
- вектор целевых значений длины `37`

После этого проверка сводится к системе:

```text
A * x = b (mod 257)
```

где `x` - это внутренняя 37-символьная часть флага.

Дальше остаётся решить систему методом Гаусса по модулю `257`.

## Восстановленный core

```text
m37474bl35_4r3_cur53d_ok_ok;;!!_ok;:!
```

## Итоговый флаг

```text
THEM?!CTF{m37474bl35_4r3_cur53d_ok_ok;;!!_ok;:!}
```

## Вывод

Главный трюк таска заключался в том, чтобы не залипнуть в фейковой VM.
Самый короткий путь решения был таким:

1. распаковать `n00t.lua`
2. распарсить Lua 5.1 bytecode
3. найти настоящий валидатор
4. свести проверку к линейной системе
5. решить её по модулю `257`
