# No 7race writeup

## Задание

Дан crypto-таск `No 7race`.

В архиве лежит один файл:

- `challenge.py`

Формат флага:

```text
THEM?!CTF{...}
```

## Читаем условие в коде

Вся логика задачи такая:

```python
flag = open("flag.txt","rb").read()
if len(flag) > 50:
    exit()

a = int.from_bytes(open("flag.txt","rb").read(), byteorder='big')

b = a << 77777
b = str(b)
if not b.endswith('03081127692533913997381228658418928780421416188103339458770036280397929450297959557812089439331054492922876854076547798835969658432397983993314299716042752'):
    exit()
```

То есть нам дают условие на десятичный суффикс числа:

```text
a << 77777
```

где `a` - это big-endian integer от байтов флага.

## Переписываем условие

Сдвиг влево на `77777` бит - это просто умножение на `2^77777`, значит:

```text
b = a * 2^77777
```

Проверка:

```text
str(b).endswith(target)
```

эквивалентна сравнению по модулю `10^155`, потому что суффикс имеет `155` десятичных цифр:

```text
b ≡ target (mod 10^155)
```

Следовательно:

```text
a * 2^77777 ≡ target (mod 10^155)
```

После этого задача превращается в чистую modular arithmetic.

## Разложение модуля через CRT

Факторизуем:

```text
10^155 = 2^155 * 5^155
```

и рассматриваем уравнение отдельно по этим модулям.

### По модулю `2^155`

Так как:

```text
2^77777
```

содержит намного больше чем `155` множителей двойки, то:

```text
2^77777 ≡ 0 (mod 2^155)
```

Значит:

```text
a * 2^77777 ≡ 0 (mod 2^155)
```

Это говорит только о том, что target тоже должен быть кратен `2^155`.
Про само `a` здесь информации почти нет.

### По модулю `5^155`

А вот здесь всё хорошо, потому что `gcd(2, 5) = 1`.
Следовательно, `2^77777` обратимо по модулю `5^155`.

Поэтому можно просто выразить:

```text
a ≡ target * (2^77777)^(-1) (mod 5^155)
```

В Python это считается напрямую:

```python
inv = pow(2, -77777, 5**155)
a = (target * inv) % (5**155)
```

## Почему решение practically единственное

Формально мы восстанавливаем `a` только по модулю `5^155`.

Но в условии дополнительно сказано, что длина флага не больше `50` байт.
То есть настоящий `a` достаточно маленький, и при переводе результата обратно в байты сразу получается нормальная ASCII-строка флага.

## Solve script

```python
target = int("03081127692533913997381228658418928780421416188103339458770036280397929450297959557812089439331054492922876854076547798835969658432397983993314299716042752")

mod5 = 5**155
inv = pow(2, -77777, mod5)
a = (target * inv) % mod5

flag = a.to_bytes((a.bit_length() + 7) // 8, "big")
print(flag.decode())
```

Результат:

```text
THEM?!CTF{NUMB3R_TH30R3M_1S_FUN}
```
